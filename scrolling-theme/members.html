Our Team

We are divided into two groups

Builders. Programmers. Both.

Builders

Builders should be familiar with all the robot parts that we have and be able to fully utilize Simple Machines like gears and pulley/chain systems. As a rule, the builders shouldnt handle the wiring, but should put the robot's parts on the robot at the very least.

Programmers

This is the most important job in the club. A Programmer must understand all parts of the functioning of the robot. Builders should communicate all details to the programmers. In a sense, the programmers are in charge maibly because a builder is useless without a programmer, but a programmer can do perfectly fine without a builder.

Our Repositories

Currently, all of our team's code is on Github in our Organization . Github is a website used by professional programmers all over the world, probably the most popular one. A repo, or repository, is a like a special folder that can be updated to the cloud, but also can be rewinded back to a checkpoint. This repo updates the way a video game is, the user decides when to save and when to rewind.


ftc_app is the main competition repo. Written in java. The programmer tutorial discusses the usage of this repo.


nxt-comp-app is another repo that contains the tools we use to test robots. It takes advantage of the old Lego NXT System that used to be used in the competition. It connects the NXT Brick to a computer and game controller, and we can use the same tools we would for the competition.
