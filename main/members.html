Builders

Builders should be familiar with all the robot parts that we have and be able to fully utilize Simple Machines like gears and pulley/chain systems. As a rule, the builders shouldnt handle the wiring, but should put the robot's parts on the robot at the very least.


  • Legal Illegal Parts List - details the stuff we can use in competition.

  • Simple Machines Webpage - holds info about simple Machines

  • Motors pdf - describes tetrix motor usage

  • Servos pdf - describes servo usage

  • Servos Webpage - describes how servos work


Programmers

This is the most important job in the club. A Programmer must understand all functions and parts of the robot. Builders should communicate all details to the programmers. In a sense, the programmers are in charge because a builder is useless without a programmer, but a programmer can do perfectly fine without a builder.

  • This tutorial we made should be enough to get you started as a programmer.


Our Repositories

Currently, all of our team's code is on Github in our Organization . Github is a website used by professional programmers all over the world. A repo, or repository, is a like a special folder that can be updated to the cloud. This repo updates the way a video game does, the user decides when to save and when to rewind.

  • ftc_app is the main competition repo. Written in java. The programmer tutorial discusses the usage of this repo.

  • nxt-comp-app is another repo that contains the tools we use to test robots. It takes advantage of the old Lego NXT System that used to be used in the competition. It connects the NXT Brick to a computer and game controller, and we can use the same tools we would for the competition.

  • FTC_5060_Archive The archives that hold all past competition code (since the start of android phone usage).
